Ultimate Guide to Creating a Minecraft Server

by N. Kostin

Originally written 18 December 2021. Last updated 24 April 2022.

Introduction

If you've ever wanted to play multi-player Minecraft, the options that most people consider are:

  • A Minecraft Realms subscription from Microsoft.
    • Pros:
      • Probably easiest to configure.
    • Cons:
      • Only for vanilla Minecraft - there's no option to run modded Minecraft.
      • Zero world customization. A Minecraft Realm generates the world from a random seed. There's no option to, say, generate a superflat world.
      • There's a relatively prohibative player limit. Only 10 players can play at a given time.
  • Server hosting, from services such as Shockbyte or Apex Hosting.
    • Pros:
      • Better customization options than with Minecraft Realms.
      • Support teams and automatic server backups.
    • Cons:
      • Becomes rather expensive quickly for anything other than the most basic plans.
  • Running a Minecraft server using a VPS (virtual private server).
    • Pros:
      • The most customization. You have the freedom to set the world seed, the max player limit, toggle PVP, and so much more.
      • Ability to run any mods, and install any modpacks.
      • Cheapest option if done correctly. Paying for the amount of bandwidth you use means that you can simply turn the server off while nobody is playing, such that excess costs are not accrued.
    • Cons:
      • Probably the most difficult to configure (hopefully less so with my guide!)

Accordingly, I maintain that by far the most extensible and cost-effective to play multi-player Minecraft is to run your own VPS.

What follows is meant to be a self-contained guide to launching and maintaining a Minecraft Java-Edition server. No pre-requisite technical knowledge and certainly no special computer hardware is needed to follow this guide.
